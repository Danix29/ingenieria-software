## About

**Asignatura:** Ingenieria del Software · UAH GII · Curso 2025-26

Software engineering principles: requirements specification, full UML modelling, software testing methodologies and quality assurance. Theory applied to real system design projects.

---

## Topics covered

| Topic | Content |
|-------|---------|
| Requirements | IEEE 830 SRS, use case documentation, functional and non-functional requirements |
| UML | Class, sequence, use-case, deployment, activity, state diagrams |
| Testing | JUnit, white box (cyclomatic complexity, basis paths), black box (equivalence classes, boundary values) |
| Design patterns | GoF patterns: Singleton, Observer, Factory, MVC |
| Software metrics | Cyclomatic complexity V(G), coupling, cohesion |
| Reverse engineering | Code to design documentation |

---

## Practices

| # | Name | Description |
|---|------|-------------|
| PECL1 | [talleres-mecanicos](./pecl1-talleres/) | Full IEEE 830 SRS, use case docs, class/sequence/deployment UML diagrams for distributed garage chain management system with VPN and MVC architecture |

## UML deliverables per practice

```
PECL1:
├── SRS (IEEE 830 format)
├── Use case diagrams + documentation
├── Class diagram
├── Sequence diagrams (1 per actor)
└── Deployment diagram

PECL2:
├── Class hierarchy (inheritance)
├── JUnit test suite
├── Cyclomatic complexity (V(G))
├── Basis paths
├── White box test cases
└── Black box test cases (equivalence partitions)
```
